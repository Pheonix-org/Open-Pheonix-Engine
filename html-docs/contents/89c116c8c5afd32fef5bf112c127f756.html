MethodHandleInlineCopyStrategy

UMLClass
UML :: openJDK :: java :: lang :: invoke :: StringConcatFactory :: MethodHandleInlineCopyStrategy

Description

<p><b>{@link Strategy#MHINLINESIZED_EXACT}: "MethodHandles inline, sized exactly".</b> <p>This strategy replicates what StringBuilders are doing: it builds the byte[] array on its own and passes that byte[] array to String constructor. This strategy requires access to some private APIs in JDK, most notably, the read-only Integer/Long.stringSize methods that measure the character length of the integers, and the private String constructor that accepts byte[] arrays without copying. While this strategy assumes a particular implementation details for String, this opens the door for building a very optimal concatenation sequence. This is the only strategy that requires porting if there are private JDK changes occur.
