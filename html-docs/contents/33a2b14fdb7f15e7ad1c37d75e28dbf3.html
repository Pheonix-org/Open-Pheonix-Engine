FieldInfo

UMLClass
UML :: openJDK :: java :: lang :: reflect :: ProxyGenerator_v49 :: FieldInfo

Description

If there is only one method with a given signature, there cannot be a conflict. This is the only case in which a primitive (or void) return type is allowed. List of return types that are not yet known to be assignable from ("covered" by) any of the others. Compare the new return type to the existing uncovered return types. If an existing uncovered return type is assignable to this new one, then we can forget the new one. If the new return type is assignable to an existing uncovered one, then should replace the existing one with the new one (or just forget the existing one, if the new one has already be put in the list). If we got through the list of existing uncovered return types without an assignability relationship, then add the new return type to the list of uncovered ones. We shouldn't end up with more than one return type that is not assignable from any of the others. A FieldInfo object contains information about a particular field in the class being generated. The class mirrors the data items of the "field_info" structure of the class file format (see JVMS 4.5).
