StartupHelper

UMLClass
UML :: opex :: backend :: runtime :: environment :: StartupHelper

Description

This class is intended for use only by OPEX. A container for organised and fragmentation of OPEX's startup routine. Subroutines are written in source following the order of expected correct execution. @author gordie

Operations

Visibility Name Description

Post engine startup routine
protected runStartupSubroutines(): void

Invokes subroutines
private addConsoleInstructions(): void

Console instruction subroutine Adds OPEX's default internal console instructions to the console.
private startRunnables(): void

Runnables subroutine Uses the thread manager to execute OPEX's internal runables, causing them to run thier own startup routines and begin to idle. This stage should be called once all resources the executables need have been loaded, and they've been configured correctly.
